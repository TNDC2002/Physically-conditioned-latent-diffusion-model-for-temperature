# Notebooks overview

This folder contains **inference**, **metric aggregation**, **end-to-end evaluation**, and **paper-style figure** notebooks. All of them assume the Python import root is the **repository root** (see working-directory notes per notebook).

**Cluster GPU / Jupyter:** see `configs/experiment/README_notebook_on_GPU.md`.

**Training weights:** use Hydra configs under `configs/experiment/` (e.g. `python src/train.py experiment=downscaling_UNET_2mT`). Checkpoints are **not** automatically placed where the inference notebooks expect them; you must copy or symlink checkpoints (or edit paths) so names match what each notebook loads.

**Standalone scripts:** there is **no** separate `src/` or `utils/` script that writes these evaluation pickles. Reproduction is via the notebooks below; `inference_full_pipeline.ipynb` is the one place that generates **all** intermediate pickle types used by the `Fig_*` notebooks (with different filenames under `outputs/latest_inference/`).

---

## Recommended workflows

### A. Split pipeline (matches older paper workflow)

1. Run **`models_inference.ipynb`** → long-form predictions pickle under `outputs/`.
2. Run **`metric_computation.ipynb`** → tabular metrics pickles under `outputs/` (and optional comparison against a reference metrics table).
3. For **`Fig_*`**: point each notebook’s `*_file_path` at your pickles **or** use the shipped files under `pretrained_models/outputs/` if present.

### B. Single notebook (rebuild evaluation artifacts)

1. Run **`inference_full_pipeline.ipynb`** once → under `outputs/latest_inference/` it writes **results**, **metrics**, **spatial errors**, **frequency distributions**, and **power spectra** pickles (see table below).  
2. Point **`Fig_*`** paths at those files if column layouts match (they are designed to mirror the shipped `pretrained_models/outputs/` schemas).

## Notebook reference

### `models_inference.ipynb`

| | |
|--|--|
| **Role** | Load multiple trained models (UNet, GAN, LDM variants, baselines), run test-set inference, append rows to a long-format `pandas.DataFrame`, save to disk. |
| **Primary inputs** | `pretrained_models/` with fixed names, e.g. `UNET_{target}.ckpt`, `GAN_{target}.ckpt`, `LDM_residual_{target}.ckpt`, `VAE_residual_{target}.ckpt`, `pde_loss_model_checkpoint.ckpt`; full `LDM-downscaling/` tree; `normalization_data.pkl`. |
| **Primary outputs** | `outputs/Our_results_trained_models_{target_var}.pkl` — long table: columns include `input_var`, `target_var`, `model`, `variable`, `spat_distr`, `min`, `max`, `time_step`. |
| **Reproduce inputs in-repo?** | **Checkpoints:** train with `configs/experiment/`, then place or symlink files under the expected `pretrained_models/` names (or edit `trained_models_path` / filenames in the notebook). **Data:** external dataset layout under `LDM-downscaling/` (not generated by this repo). **No** dedicated `.py` exporter besides this notebook. |

---

### `metric_computation.ipynb`

| | |
|--|--|
| **Role** | Read inference results; compute per-timestep metrics vs **COSMO-CLM**; optionally extra PDE / physics metrics in later cells; save metrics tables and some plots. |
| **Primary inputs** | Results pickle (default `../outputs/Our_results_trained_models_2mT.pkl` when cwd is `notebooks/`). **Also** loads `../pretrained_models/outputs/metrics_trained_models.pkl` for comparison — **remove or replace** this if you have no `pretrained_models` tree. |
| **Primary outputs** | e.g. `../outputs/Our_inference_metrics.pkl`, `Our_inference_metrics_with_PDE.pkl`, `Our_inference_metrics_with_PDE_Phys.pkl` (depending on cells run) — columns: `model`, `target_var`, `var`, `metric`, `value`. |
| **Reproduce inputs in-repo?** | **Results pickle:** yes — from `models_inference.ipynb` (same column semantics) or from `inference_full_pipeline.ipynb` as `results_latest_{target}.pkl` if you align `results_file_path` and ensure `model` / `COSMO-CLM` rows exist as expected. **Reference metrics pickle:** not produced elsewhere in this codebase; ship your own or skip that part of the notebook. |

---

### `inference_full_pipeline.ipynb`

| | |
|--|--|
| **Role** | One path from checkpoints + data to **all** evaluation pickles and inline plots (metrics, spatial mean error, histograms, spectra). Uses `metadata_test_paper_sample.csv` for the test loader by default. |
| **Primary inputs** | Same checkpoint directory convention as `models_inference.ipynb` under `pretrained_models/`; `LDM-downscaling/`; `normalization_data.pkl`; configurable `target_var`, `denoising_steps`, `output_path` (default `outputs/latest_inference/`). |
| **Primary outputs** | Under `outputs/latest_inference/` (default): `results_latest_{target_var}.pkl`, `metrics_latest_{target_var}.pkl`, `spatial_errors_latest_{target_var}.pkl`, `freq_distribution_latest_{target_var}.pkl`, `power_spectra_latest_{target_var}.pkl`; plus PNGs from `show_*` helpers in that folder. |
| **Reproduce inputs in-repo?** | Same as `models_inference.ipynb` for checkpoints and data. **This notebook is the in-repo way to regenerate the same *kinds* of pickles** the `Fig_*` notebooks expect (different basenames). |

---

### `models_inference_executed.ipynb`

| | |
|--|--|
| **Role** | Same logical flow as `models_inference.ipynb`, kept as an executed / archival copy for inspection. |
| **Inputs / outputs** | Same pattern as `models_inference.ipynb`; prefer the non-`_executed` notebook for new runs. |

## `Fig_*` notebooks (plotting only)

Defaults assume paths under **`pretrained_models/outputs/`** (release bundle). To use **your** artifacts, change the `*_file_path` variables at the top of each notebook to match files from **`inference_full_pipeline.ipynb`** (or any pickle with the **same columns**).

| Notebook | Expected pickle content (schema) | Default path (from `notebooks/` cwd) | In-repo producer with same structure |
|----------|----------------------------------|----------------------------------------|--------------------------------------|
| `Fig_metrics.ipynb` | `pandas.DataFrame`: `model`, `target_var`, `var`, `metric`, `value` | `../pretrained_models/outputs/metrics_trained_models.pkl` | `metric_computation.ipynb` → `Our_inference_metrics*.pkl`; or `inference_full_pipeline.ipynb` → `metrics_latest_*.pkl` |
| `Fig_spatial_errors.ipynb` | Rows: `variable`, `model`, `spat_distr` (mean error field), `min`, `max` | `../pretrained_models/outputs/spatial_errors_trained_models.pkl` | **`inference_full_pipeline.ipynb`** → `spatial_errors_latest_*.pkl` |
| `Fig_power_spectra.ipynb` | Rows: `model`, `variable`, `spectra`, `fft_freq` | `../pretrained_models/outputs/power_spectra_trained_models.pkl` | **`inference_full_pipeline.ipynb`** → `power_spectra_latest_*.pkl` |
| `Fig_frequency_distrib.ipynb` | Rows: `model`, `variable`, `x_s`, `freq_distr` | `../pretrained_models/outputs/freq_distribution_trained_models.pkl` | **`inference_full_pipeline.ipynb`** → `freq_distribution_latest_*.pkl` |
| `Fig_snapshots.ipynb` | Long **results** table (same family as inference output): per timestep / model / variable fields including `spat_distr` | `../pretrained_models/outputs/results_trained_models_2mT.pkl` (and UV variant) | **`models_inference.ipynb`** → `Our_results_trained_models_*.pkl`; or **`inference_full_pipeline.ipynb`** → `results_latest_*.pkl` |

**Note:** `metric_computation.ipynb` does **not** write spatial-error, spectrum, or frequency pickles; only **`inference_full_pipeline.ipynb`** (among files in this repo) generates those alongside the results pickle.
